1: Assign yourself to a task in GitHub issues
2: pull from origin main 
3: make new task/feature branch and switch to it; call it "issue<issue_number>" if its related to an issue.
4: pull from main branch
5: make changes (whenever possible; make new functions/classes instead of adding to existing ones)
6: commit changes to feature branch 
7: repeat 4 and 5 until feature is done.
8: when feature is done and tested push it to origin
9: Create pull request
10: Switch and pull from main 
11: Merge feature into main (git merge feature-branch)
12: push main to origin main
